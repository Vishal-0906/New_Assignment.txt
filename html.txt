1. HTML Basics
Theory Assignment

Question 1: Define HTML. What is the purpose of HTML in web development?
Answer:
HTML (HyperText Markup Language) is the standard language used to create and structure content on the web.
Purpose:
Defines the structure of web pages.
Displays text, images, videos, and links.
Provides the foundation for styling (CSS) and interactivity (JavaScript).
Ensures content is readable by browsers and search engines.


Question 2: Explain the basic structure of an HTML document. Identify the mandatory tags and their purposes.
Answer:
<!DOCTYPE html>          <!-- Declares document type -->
<html>                   <!-- Root element -->
<head>                   <!-- Contains metadata like title, links, etc. -->
  <title>My Webpage</title>
</head>
<body>                   <!-- Visible page content -->
  <h1>Welcome</h1>
  <p>This is my webpage.</p>
</body>
</html>

Mandatory tags:
<!DOCTYPE html> → defines HTML5 document.
<html> → root element.
<head> → metadata (title, CSS, etc.).
<body> → visible content.

Question 3: What is the difference between block-level elements and inline elements in HTML? Provide examples.
Answer:
Block-level elements: Start on a new line and take full width.
Examples: <div>, <p>, <h1>–<h6>, <section>, <article>.
Inline elements: Do not start on a new line; take only required width.
Examples: <span>, <a>, <strong>, <em>, <img>.

Question 4: Discuss the role of semantic HTML. Why is it important for accessibility and SEO? Provide examples.
Answer:
Semantic HTML uses meaningful tags that describe content.
Importance:
Helps screen readers and assistive technologies understand the page.
Improves SEO by giving search engines clear structure.
Examples:
<header>, <footer>, <article>, <section>, <nav>, <main>.

2. HTML Forms

Theory Assignment

Question 1: What are HTML forms used for? Describe the purpose of input, textarea, select, and button elements.
Answer:
HTML forms are used to collect user input and send it to the server.
<input> → single-line input (text, email, password, etc.)
<textarea> → multi-line input (like messages)
<select> → dropdown menu for options
<button> → submits or triggers form actions

Question 2: Explain the difference between the GET and POST methods in form submission.
Answer:
GET:
Data is sent via URL.
Visible in address bar.
Used for search queries or non-sensitive data.
POST:
Data is sent in the request body.
More secure, not visible in URL.
Used for sensitive or large data.

Question 3: What is the purpose of the label element in a form, and how does it improve accessibility?
Answer:
The <label> element associates text with form controls, improving accessibility.

Screen readers can identify input purpose.

Clicking the label focuses the input.
Example:
<label for="email">Email:</label>
<input type="email" id="email" name="email">

3. HTML Tables

Theory Assignment

Question 1: Explain the structure of an HTML table and the purpose of each element.
Answer:
<table>
  <thead>   <!-- Table header -->
    <tr>     <!-- Table row -->
      <th>Column Name</th> <!-- Table header cell -->
    </tr>
  </thead>
  <tbody>   <!-- Table body -->
    <tr>
      <td>Data Cell</td> <!-- Table data cell -->
    </tr>
  </tbody>
</table>

Question 2: What is the difference between colspan and rowspan?
Answer:
colspan: Merges multiple columns horizontally.
Example: <td colspan="2">
rowspan: Merges multiple rows vertically.
Example: <td rowspan="3">


Question 3: Why should tables be used sparingly for layout purposes? What is a better alternative?
Answer:

Tables are meant for tabular data, not layout. Using tables for layout causes accessibility and responsiveness issues. Better alternative: Use CSS Grid or Flexbox for layout.
